Introduction

Welcome to MAME4droid (Current)

MAME4droid (Current) is developed by David Valdeita (Seleuco) as a port of the latest desktop MAME emulator by MAMEDev and contributors. It emulates arcade games and systems supported by this version of MAME which includes over 40000 different systems.

This version of MAME4droid was designed to be used with high-end Android devices because it is based on the PC MAME version which requires higher specifications than the older versions.

Even with a high end device, do not expect "modern" arcade games from the 90s and beyond to necessarily work at full speed or compatibility.

With over 40000 systems supported, some games will run better than others; some games may not run at all. It is impossible to support such a vast number of titles, so please do not email the developer asking for support for a specific game.

If you have a lower end Android device, you will find MAME4droid (0.139u1) to be much faster - but it supports fewer, older ROMs and more games will require sound samples for a complete audio experience.

MAME4droid will do nothing without having at least one compatible ROM to emulate. Your first step should be to obtain compatible ROM files and place them in the designated ROMs folder (see Installation and Supported Games for more information). Technical Note: this version includes MAME's UML-based ARM64 dynamic recompiler (DRC), added back in v1.21. Instead of interpreting a game's CPU instructions one by one (extremely slow for high-spec arcade CPUs), the DRC translates them into native ARM64 machine code on the fly, running directly on your device's own CPU. This is what makes MIPS/ARM/SH2/PowerPC-based games (which used to be unplayable, or only usable at a fraction of real speed, before v1.21) actually playable now — Killer Instinct (which runs on a MIPS R4600 main CPU) is a good example of a game that goes from choppy/unplayable without the recompiler to full speed with it. This is one of the biggest advantages this version has over the older MAME4droid (0.139u1) mentioned above, which never had a dynamic recompiler at all.
