Installation

First Run: Where to Store Your ROMs

The very first time you start MAME4droid, it asks you where you want to keep your ROM files (and the rest of MAME's data: artwork, samples, save states, etc). You get two choices:

  • Internal: MAME4droid creates an empty folder inside its own private app storage and uses that. You then copy your ROMs into it yourself, either from a PC over USB/ADB, or with a file manager app that can browse the app's private folder. This data is deleted automatically when you uninstall the app.
  • External Storage: you pick an existing folder anywhere on your device (an SD card, a folder you already use for ROMs, a cloud-synced folder, etc.) using Android's standard folder picker. MAME4droid asks Android for permission to read/write that one folder and remembers it for next time. This data survives uninstalling the app, and you can freely move files in and out of it with any regular file manager, no PC needed.

Important: with External Storage, the folder you pick is your ROMs folder itself — MAME4droid does not create a roms subfolder inside it. Put your ZIPped ROM files directly in the folder you picked. Inside that same folder, you can additionally create these optional subfolders yourself:

artwork/ for your bezel/marquee artwork sets
samples/ for sound samples
software/ for software-list media (cassette/disk images used by non-arcade systems with removable software, like 8-bit home computers)
sta/ for save states, only if you also enable Save States in ROMs Path (see below)

Everything else (MAME configuration, hiscores, NVRAM, snapshots) always stays in MAME4droid's own internal data, regardless of where your ROMs live.

With Internal storage instead, MAME4droid creates the full folder tree below (including a roms/ subfolder) inside its own private app storage.

About the folder picker (Storage Access Framework)

Since Android 10, apps are no longer allowed to freely read and write anywhere on shared storage — they only get access to folders the user explicitly grants through Android's own picker (this is called the Storage Access Framework, or SAF). That's the picker that opens when you choose External Storage above, or when you change your ROMs folder later. It's a bit slower than direct file access, but MAME4droid caches file listings so it stays fast in day-to-day use once your ROMs are indexed.

Tip: keep your ROMs folder flat (no ROMs buried in subfolders) and don't mix in unrelated, unzipped files — the folder should really only contain your ZIPped MAME ROM sets.

Changing Your ROMs Location Later

You don't have to get it right the first time. Go to Options->Settings->General->Change ROMs Path to run that same Internal/External Storage choice again (the app restarts afterwards). If you're switching to a new external folder, remember to actually move or copy your ROM files there first.

If you want your save states to live alongside your ROMs on external storage (so Android doesn't clean them up when you eventually uninstall), enable Options->Settings->General->Save States in ROMs Path.

Advanced: Relocating the Whole Installation

Separately from your ROMs folder, experienced users can relocate MAME4droid's entire data folder (not just ROMs, but every configuration/support folder listed below) to a specific path under Options->Settings->Advanced->Installation Path. This one is typed in manually rather than picked, and is intended for advanced use only: entering a valid but unintended path will make MAME4droid treat it as a brand new, empty installation. Leave this alone unless you specifically need it.

Folder Layout

This is MAME4droid's internal data folder tree (used as-is with Internal storage; with External Storage only roms/artwork/samples/software below actually live in your picked folder, as explained above, and the folder names there don't have the trailing slash). It's the same layout desktop MAME uses, so you can reuse existing artwork/samples collections. Check the Resources and Assets section for where to find these assets, and Supported Games for ROM naming rules.

artwork/ -> Artwork directory
cfg/ -> MAME configuration files directory
hi/ -> Hiscores directory
nvram/ -> NVRAM files directory
roms/ -> ROMs directory
samples/ -> Samples directory
snap/ -> Screen snapshots directory
sta/ -> Save states directory
