ネットプレイ

ネットプレイは、2 台のデバイスが Wi-Fi/ネットワーク経由で同じゲームを一緒にプレイできる機能で、 一方がホスト、もう一方がクライアントとして動作します。両方のデバイスが まったく同じ ROM を同時に実行し、コントローラーの入力だけがネットワークを流れます。

モード

ホストのみが、ゲームを作成する前にネットプレイダイアログからモードを選びます。 クライアントは何も選ぶ必要はありません：接続するとすぐにホストが使っているモードを自動的に採用します。

  • ロックステップ：最も安全で互換性の高いモード。両方のデバイスが 1 フレーム進める 前に相手の入力を待つので、ゲームは常に完璧に同期します。トレードオフは入力遅延です：接続が悪い （ping が高い）ほど、ボタンを押してからそれが起こるまでの遅延を強く感じます。すべてのゲームで 動作します。
  • ロールバック：低遅延モードで、素早い反応が必要なゲーム（NeoGeo、CPS1/CPS2、 格闘ゲームなど）に最適です。各デバイスは相手の入力を予測してすぐにプレイを続けます。実際の入力が 届いたときに予測が外れていた場合、数フレームを静かに巻き戻し、正しい入力で再シミュレートします。 ほとんどの場合、遅延はまったく感じません。ロックステップより CPU/バッテリーを多く消費し、巻き戻す ためのステートセーブの短い履歴を保持する追加の RAM を必要とします。

入力遅延

どちらのモードも、ローカル入力を即座に適用するのではなく、数フレーム遅らせてから有効にすること （対応する相手の入力と同時に届くように）で、ネットワーク遅延を隠します。デフォルトでは自動 に設定されています：MAME4droid が接続の ping を測定して自分で遅延を調整するので、何も触る必要はありません。 モードによって少し動作が異なります：

  • ロールバックでは、自動は安全な範囲内で往復時間の約半分を継続的に追跡します — ロールバック自身の予測/巻き戻しが遅延のカバーしきれない分を吸収するので、低く保たれます。
  • ロックステップでは、自動は時折段階的に調整し（両方のデバイスで同じフレームに 当たるように計画される）、小さな ping の変動ごとではなく、持続的な遅延の変化に反応します。

自動のままにしたくない場合は、オプション->設定->ネットプレイ->ネットプレイ遅延で、 特定の値（1～6 フレーム）に固定できます。モードと同様に、これは実質的にホストが決めます： 接続後、クライアントはホストがそのセッションで最終的に使う遅延を採用します。

ロールバックの互換性と自動フォールバック

ロールバックはマシンの完全な状態を毎秒何度も保存/復元する必要があるため、すべてのゲームに等しく 適しているわけではありません。MAME4droid はロールバックセッションの開始時にこれを自動的にチェックし、 必要に応じてロックステップにフォールバックします（画面上の警告付き）— あなたは何もする必要は ありません：

  • ステートセーブが大きすぎる：RAM の多いゲーム（例：ハードディスク/CHD を使うもの） は、メモリに多くのコピーを保持するには大きすぎるステートセーブを生成します。設定された RAM 予算に 収まらない場合、そのセッションは自動的にロックステップで実行されます。
  • デバイス間でステートのレイアウトが異なる：2 台のデバイスはドライバーに対して まったく同じ内部セーブデータを登録する必要があります（同じ MAME4droid ビルド/バージョン）。ゲーム 状態を交換する前に不一致が検出された場合、両者は静かな非同期のリスクを冒すのではなくロックステップ にフォールバックします。

ロールバックで問題が知られているゲーム

一部のゲームは、アナログコントロール（ステアリングホイール、スピナー/パドル、ペダル）を、フレーム ごとに 1 回ではなくハードウェアタイミングの精度でポーリングする副次的なチップ（保護 MCU または 2 つ目の サウンド CPU）を通じて読み取ります。ロールバックの巻き戻して再生する処理は、元の実行とはわずかに異なる アナログ読み取り値を捕捉することがあり、これらのゲームで時折の非同期として現れることがあります：

  • Arkanoid（およびそのパドル + 保護 MCU ハードウェアを共有するクローン/海賊版）
  • Buggy Boy / Buggy Boy Jr. / TX-1 系列（ステアリングホイール + ペダルが 2 つ目の サウンド CPU によって読み取られる）

このようなゲームで非同期に気づいたら、代わりにロックステップを選んでください — ロックステップは決して巻き戻したり再シミュレートしたりしないので、この種の問題はそこでは起こりません。 同様のハードウェア（通常のフレームごとの入力パスの外で読み取られるアナログダイヤル/パドル/ホイール/ ペダルのコントロール）を持つ他のゲームも同じ動作を示す可能性があります。

接続統計オーバーレイ

セッションがアクティブな間、小さな ping/遅延インジケーターを画面下部に任意で表示できます：現在の モード、ping（往復時間）、入力遅延を、良好な接続には緑、目立つ接続には黄、 悪い接続には赤で色分けします。数秒ごとに更新され、セッションが終わると自動的に消えます。 オプション->設定->ネットプレイ->接続統計オーバーレイを表示でオンにします（デフォルトは オフ）。

非同期検出器

デフォルトでは、ロールバックセッションは両方のデバイス間で定期的な状態のチェックサムを比較する バックグラウンドチェックも実行します。2 つの画面が本当にずれてしまったことを検出すると、現在のゲームが ロールバックに適さない可能性があることを伝える短い赤い警告を表示します — それが再同期 を試すか、そのセッションをロックステップに切り替える合図です。このチェックは毎フレーム 少しの追加 CPU を消費します。古いデバイスでそのコストを節約したく、警告を諦めても構わないなら、 オプション->設定->ネットプレイ->非同期検出器でオフにしてください。

ゲームを作成する

  • ネットプレイダイアログを開き、ゲームを作成をタップします。
  • このセッションのロックステップまたはロールバックを選びます。
  • MAME4droid はリスニングポートを開き（オプション->設定->ネットプレイポートで設定可能、 デフォルト 2080）、待機画面にあなたのアドレスが用途とともに表示されます：同じ Wi-Fi/ネットワーク上 のプレイヤー向けのローカル IP、インターネット経由で参加するプレイヤー向けのグローバル IP — 共有ボタンでそれらを任意のメッセンジャーで送信でき、相手プレイヤーの 参加欄にそのまま貼り付けられます。
  • プレイしたいゲームをロード/選択します。クライアントが接続すると、両方のデバイスで自動的に起動され ます。

ゲームに参加する

  • ネットプレイダイアログを開き、ゲームに参加をタップします。
  • ホストの IP アドレスを入力します（両方が同じポートを使っていることを確認してください）。
  • それだけです — 選ぶモードはありません。接続が確立されるとすぐに、あなたのデバイスはホストの モードを採用し、ホストが選択したゲームを自動的にロードします。

注意：両方のデバイスは同じネットワーク上で到達可能である必要があり（同じ Wi-Fi、 または選んだポートがそれらの間で開いているようにルーティングされている）、まったく同じ ROM ファイルが 必要で、同じ MAME4droid バージョンを実行しているべきです — バージョンの不一致は検出され、静かな 非同期のリスクを冒す代わりに、警告とともに接続が拒否されます。

インターネット対戦（サーバー不要）

異なるネットワーク上の 2 人のプレイヤーは、任意のメッセンジャーでアドレスを交換することで直接接続 できます — アカウント、ロビー、サーバーは不要です。ホストの作成は LAN とまったく同じです： ゲームを作成をタップして待機画面を読んでください — 何を共有し、次に何をするかを 教えてくれます。

通常のケースは自動です（UPnP）：ホストをしている間、MAME4droid はルーターにネット プレイポートを自分で開くよう要求します。待機画面にルーターのポートがマッピングされました（UPnP） と表示されたら、共有をタップしてアドレスを相手プレイヤーに送るだけです：相手は ゲームに参加をタップしてインターネットのアドレスを貼り付けます。それだけです — 相手プレイヤーがモバイルデータでも動作します。マッピングはセッション終了時に削除されます。この機能は オプション->設定->ネットプレイ->UPnP ポートマッピングでオフにできます。

UPnP が利用できない場合（その行が表示されない、または参加が成立しない場合）、 MAME4droid は両方のルーターを通じて「ホールパンチング」を行うこともできます。パンチングにはアドレスを 両方向で交換する必要があります：

  • ステップ 1（ホスト）：待機画面で共有をタップし、アドレスを相手 プレイヤーに送ります。
  • ステップ 2（クライアント）：ホストのグローバルアドレスに参加します。次に接続画面で 共有をタップして、自分のグローバルアドレスをホストに送り返します。
  • ステップ 3（ホスト）：待機画面で相手 IPをタップし、クライアントの アドレスを貼り付けます。接続は通常数秒以内に成立します。

要件と制限：

  • ほとんどの家庭用 Wi-Fi ルーター間で動作します。モバイルデータ （4G/5G）ではホールパンチングは通常失敗します：キャリアの CGNAT は接続ごとにグローバルポートを 変える傾向があります — 対称型の警告が表示されなかった場合でもです。モバイルデータのプレイ ヤーは、ポートが転送されたホストには参加できます（上記の UPnP、または下記の手動方法）。
  • 対称型 NAT を検出しましたと表示された場合、インターネット対戦は失敗する可能性があります — とにかく試すか、ポート転送の方法を使ってください。
  • 厳格なルーターやゲーミングルーター（SPI ファイアウォール — 例：ASUS TUF/ROG）は、両方の NAT が問題なさそうに見えても、パンチした接続を静かに破棄することがあります。ルーター設定に NAT タイプのオプションがあればFull Coneを選んでください。なければポート転送の方法を使って ください。
  • 両方のプレイヤーが CGNAT の背後にいる場合（どちらの端にも到達可能なグローバル IPv4 がない）、ホールパンチングもポート転送も機能しません。これは ISP の制限であり、アプリの バグではありません。ISP にグローバル IPv4 を要求するか、それを持つネットワークでプレイしてください。
  • LAN より高い ping を予想してください — ロールバックモードを強く推奨します。
  • 両方のデバイスが同じ Wi-Fi にある場合は、グローバルアドレスではなく通常のローカルフロー （ローカル IP）を使ってください。

常に機能する方法：ホストはルーターで UDP ポート 2080 を自分のデバイスに転送し （ルーターはこれをバーチャルサーバー、ポート転送、またはカスタムの Open NAT ゲームプロファイルと 呼びます）、グローバル IP を共有します。クライアントは単にそれに参加します。これが必要なのはホストの ルーターだけです — モバイルデータのクライアントもプレイできるようにします。これは UPnP がオフまたは 壊れているルーター向けの、UPnP の手動版です。

IPv6（ネットワークプロトコル）

オプション->設定->ネットプレイ->ネットワークプロトコルで接続方式を 選択します：IPv4（デフォルト）、IPv6、または 自動（IPv6 + IPv4）。IPv6 には NAT がなく、各デバイスが固有のグローバル アドレスを持つため、インターネット対戦は通常直接接続できます — ポート開放も UPnP も 不要で、上記の CGNAT の制限もなくなります。特に IPv6 付きモバイルデータ （4G/5G）の 2 台のスマートフォン — ほとんどのキャリアが提供 — は ルーター設定なしで直接つながります。インターネット対戦の最も簡単で確実な方法です。 デフォルトは最も実績のある IPv4 ですが、自動も 試す価値があります：どの相手とも対戦でき、最適な経路を自動で選びます。

  • IPv4：上記で説明した従来の動作です。
  • IPv6：両者が IPv6 のみで接続します。共有される招待には [角括弧] 付きのアドレスが含まれます。共有と貼り付けの使い方は まったく同じです。
  • 自動：ホストは両方のプロトコルを受け付け、持っているすべての アドレス（IPv6、IPv4、ローカルネットワーク）を共有します。クライアントは自分の 回線で使えるものを自動的に選びます。相手の環境が不明な場合はこれを選んで ください。

両者は同じ設定を使うべきです（自動はどちらとも組み合わせられます）。一部のルーターは 着信をブロックする IPv6 ファイアウォールを備えています。内蔵のホールパンチングが通常は 自動で開きますが、開かない場合はそのファイアウォールでネットプレイのポートを許可する ことが IPv6 版のポート開放になります。

ルーターの IPv6 が有効でも ISP がパブリック IPv6 プレフィックスを配布しない場合、 端末には同一ネットワーク専用の (ULA) アドレスが割り当てられます。インターネット対戦は できませんが、同一ネットワーク内の 2 台なら IPv6 で対戦できます。ホストはそのアドレスを 同一ネットワークのみと表示します。

ゲーム中

  • 再同期：進行中のロールバックセッションでは、ネットプレイダイアログに 再同期ボタンがあります。ホストが自分の正確なゲーム状態をクライアントに再送する間、 両方のデバイスを一瞬フリーズさせ、両マシンを同期に戻します。2 つの画面が目に見えてずれたことに 気づいたら使ってください。ロックステップではそもそもずれないので、必要ありません（表示もされません）。
  • 切断：両方のデバイスのネットプレイセッションを終了し、通常のオフラインプレイに 戻ります。

ヒント

  • 安定した Wi-Fi 接続は生の速度よりはるかに重要です — 安定した 2.4GHz リンクは、混雑した 5GHz よりうまく動作することがよくあります。
  • セッション中は両方のデバイスの画面をオンにし、アプリを前面に保ってください。バックグラウンドに すると接続が一時停止することがあります。
  • 特定のゲームでロールバックがカクつく場合は、そのセッションをロックステップに切り替えてみて ください — 一部のハードウェア（上記参照）はロールバックに向いていません。
