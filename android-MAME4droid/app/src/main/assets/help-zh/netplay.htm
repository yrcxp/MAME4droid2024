联机对战

联机对战让两台设备通过 Wi-Fi/网络一起玩同一个游戏，一台作为 主机，另一台作为 客户端。两台设备同时运行完全 相同的 ROM；只有手柄输入通过网络传输。

模式

只有 主机 在创建游戏前从联机对战对话框中选择模式。客户端无需 选择任何内容：连接后会自动采用主机使用的模式。

  • 锁步：最安全、最兼容的模式。两台设备在推进一帧前都会等待 对方的输入，因此游戏始终完美同步。代价是输入延迟：连接越差（ping 越高）， 从按下按钮到看到反应之间的延迟就越大。适用于所有游戏。
  • 回滚：低延迟模式，适合快速反应的游戏（NeoGeo、CPS1/CPS2、 格斗游戏……）。每台设备预测对方的输入并立即继续游戏；如果真实输入到达时预测 结果错误，它会静默回退几帧并用正确的输入重新模拟。大多数时候您完全感觉不到 延迟。比锁步消耗更多 CPU/电池，并需要额外 RAM 来保存一小段可回退的存档历史。

输入延迟

两种模式都通过将本地输入延迟几帧后再生效（使其与对应的对方输入同时到达）来隐藏 网络延迟，而不是立即应用。默认设置为 自动：MAME4droid 测量您连接的 ping 并自行调整延迟，因此您无需操作任何内容。它根据模式略有不同：

  • 在 回滚 中，自动会在安全范围内持续跟踪约一半的往返时间 — 保持较低，因为回滚自身的预测/回退会吸收延迟未覆盖的部分。
  • 在 锁步 中，自动会偶尔分步调整（计划落在两台设备的同一帧上）， 对持续的延迟变化做出反应，而非每一次小的 ping 波动。

如果您不想保持自动，选项->设置->联机对战->联机对战延迟 可让您 将其固定为特定值（1 到 6 帧）。与模式一样，这实际上由 主机 决定： 连接后，客户端会采用主机最终为该会话使用的延迟。

回滚兼容性与自动回退

回滚需要每秒多次保存/恢复完整的机器状态，因此并非同样适合所有游戏。MAME4droid 在 回滚会话开始时自动检查，并在需要时回退到锁步（附带屏幕警告）— 您无需做任何事：

  • 存档状态过大：RAM 很大的游戏（例如使用硬盘/CHD 的游戏）产生的 存档状态太大，无法在内存中保存多份副本。如果不符合配置的 RAM 预算，该会话将 自动以锁步运行。
  • 设备间存档布局不同：两台设备必须为驱动注册完全相同的内部保存 数据（相同的 MAME4droid 版本/构建）。如果在交换游戏状态前检测到不匹配，双方将 回退到锁步，而非冒静默不同步的风险。

已知在回滚中有问题的游戏

少数游戏通过辅助芯片（保护 MCU 或第二个声音 CPU）读取模拟控制（方向盘、旋钮/桨、 踏板），该芯片以硬件时序精度而非每帧一次进行轮询。回滚的回退并重放随后可能捕获与原始 运行略有不同的模拟读数，这在这些游戏中可能表现为偶尔的不同步：

  • Arkanoid（以及共享其桨 + 保护 MCU 硬件的克隆/盗版）
  • Buggy Boy / Buggy Boy Jr. / TX-1 系列（方向盘 + 踏板由第二个 声音 CPU 读取）

如果您在此类游戏中注意到不同步，请为其选择 锁步 — 锁步从不 回退或重新模拟，因此这类问题在那里不会发生。其他具有类似硬件（在正常每帧输入路径之外 读取的模拟拨盘/桨/方向盘/踏板控制）的游戏可能表现出相同行为。

连接统计叠加层

会话进行时，可选择在屏幕底部显示一个小型 ping/延迟指示器：当前模式、ping（往返时间） 和输入延迟，良好连接显示 绿色，明显连接显示 黄色，较差 连接显示 红色。它每几秒刷新一次，会话结束后自动消失。在 选项->设置->联机对战->显示连接统计叠加层 中开启（默认关闭）。

不同步检测器

默认情况下，回滚会话还会运行一个后台检查，比较两台设备之间的周期性状态校验和。如果它 发现两个画面确实已经偏离，会显示一个简短的红色警告，告诉您当前游戏可能不适合回滚 — 这是您尝试 重新同步 或将该会话切换到 锁步 的信号。该检查 每帧消耗少量额外 CPU；如果您希望在旧设备上节省该开销并愿意放弃警告，请在 选项->设置->联机对战->不同步检测器 中将其关闭。

创建游戏

  • 打开联机对战对话框并点击 创建游戏。
  • 为此会话选择 锁步 或 回滚。
  • MAME4droid 打开一个监听端口（可在 选项->设置->联机对战端口 中配置， 默认 2080），等待屏幕显示您的地址，每个都标有用途：您的本地 IP 用于同一 Wi-Fi/网络上的玩家，您的公网 IP 用于通过互联网加入的玩家 — 分享 按钮通过任意即时通讯软件发送它们，可直接粘贴到对方玩家的 加入框中。
  • 加载/选择您想玩的游戏；客户端连接后，它将在两台设备上自动启动。

加入游戏

  • 打开联机对战对话框并点击 加入游戏。
  • 输入主机的 IP 地址（并确保双方使用相同的端口）。
  • 就这样 — 无需选择模式。连接建立后，您的设备会立即采用主机的模式并自动加载 主机选择的游戏。

注意：两台设备需要在同一网络上可达（同一 Wi-Fi，或以所选端口在它们 之间开放的方式路由），需要完全相同的 ROM 文件，并应运行相同的 MAME4droid 版本 — 版本不匹配会被检测到，连接会被拒绝并附带警告，而非冒静默不同步的风险。

互联网对战（无需服务器）

不同网络上的两名玩家可以通过任意即时通讯软件交换地址直接连接 — 无需账户、大厅 或服务器。创建游戏与在 LAN 上完全相同：点击 创建游戏 并阅读等待屏幕 — 它会告诉您分享什么以及接下来做什么。

通常情况是自动的（UPnP）：在您作为主机时，MAME4droid 会请求您的路由器 自行打开联机对战端口。如果等待屏幕显示 路由器端口已映射（UPnP），只需点击 分享 并将您的地址发送给对方玩家：他们点击 加入游戏 并 粘贴互联网地址。就这样 — 即使对方玩家使用移动数据也能工作。映射会在会话结束时移除； 该功能可在 选项->设置->联机对战->UPnP 端口映射 中关闭。

如果 UPnP 不可用（该行未出现，或加入无法建立），MAME4droid 仍可通过 两个路由器进行「打洞」。打洞需要双向交换地址：

  • 第 1 步（主机）：在等待屏幕点击 分享 并将您的地址 发送给对方玩家。
  • 第 2 步（客户端）：加入主机的公网地址；然后在连接屏幕点击 分享 将您自己的公网地址发回给主机。
  • 第 3 步（主机）：在等待屏幕点击 对方 IP 并粘贴客户端 的地址。连接通常在几秒内建立。

要求和限制：

  • 在大多数 家用 Wi-Fi 路由器之间有效。通过 移动数据 （4G/5G）打洞通常失败：运营商 CGNAT 往往每次连接更改公网端口 — 即使未显示 对称型警告。使用移动数据的玩家 仍可 加入端口已转发的主机（上面的 UPnP， 或下面的手动方法）。
  • 如果出现 检测到对称型 NAT，互联网对战可能失败 — 无论如何都可以尝试， 或使用端口转发方法。
  • 严格或游戏路由器（SPI 防火墙 — 例如 ASUS TUF/ROG）即使两端 NAT 看起来都正常， 也可能静默丢弃打洞连接。如果路由器设置提供 NAT 类型选项，请选择 Full Cone； 否则使用端口转发方法。
  • 如果 两名 玩家都在 CGNAT 之后（两端都没有可达的公网 IPv4），打洞和 端口转发都无法工作。这是 ISP 的限制，而非应用的缺陷；请向您的 ISP 索取公网 IPv4 或 在有公网 IPv4 的网络上游玩。
  • 预计 ping 会比 LAN 高 — 强烈推荐回滚模式。
  • 如果两台设备在 同一 Wi-Fi 上，请使用正常的本地流程（本地 IP），而非公网 地址。

始终有效的方法：主机在其路由器上将 UDP 端口 2080 转发到其设备（路由器 称之为虚拟服务器、端口转发，或自定义的 Open NAT 游戏配置）并分享其公网 IP；客户端 直接加入即可。只有主机的路由器需要这样 — 它也让使用移动数据的客户端能够游玩。这是 UPnP 的手动等效方法，适用于 UPnP 关闭或损坏的路由器。

IPv6（网络协议）

选项->设置->联机对战->网络协议选择设备的连接方式： IPv4（默认）、IPv6 或 自动（IPv6 + IPv4）。IPv6 没有 NAT：每台设备都有自己的公网地址， 因此互联网对战通常可直接连接 — 无需端口转发，无需 UPnP，上文提到的 CGNAT 限制也不复存在。特别是两部使用移动数据（4G/5G）且有 IPv6 的手机 — 大多数运营商都提供 — 可以直接互联，完全无需设置路由器：这是互联网对战 最简单、最可靠的方式。默认值为 IPv4，因为它是经过最多实测的路径， 但值得试试自动：它与任何对手都兼容，并自行选择最佳路线。

  • IPv4：上文描述的经典行为。
  • IPv6：双方仅通过 IPv6 连接。分享的邀请包含一个 [方括号] 地址；分享和粘贴的用法完全相同。
  • 自动：主机同时接受两种协议，并分享其所有地址（IPv6、IPv4 和本地网络）；客户端自动选择适合自己连接的地址。不确定对方用什么时选它。

双方应使用相同的设置（自动与任何一方都能配对）。部分路由器自带会拦截入站连接的 IPv6 防火墙：内置的打洞机制通常会自行打开；如果不行，在该防火墙中放行联机对战端口 就是 IPv6 版的端口转发。

如果路由器启用了 IPv6 但运营商没有分配公网 IPv6 前缀，设备仍会获得一个仅限本地网络的 (ULA) 地址：无法通过互联网联机，但同一网络内的两台设备仍可通过 IPv6 联机。主机会将该地址 标注为仅同一网络。

游戏过程中

  • 重新同步：在进行中的回滚会话中，联机对战对话框提供一个 重新同步 按钮。它强制两台设备短暂冻结，同时主机将其确切的游戏状态 重新发送给客户端，使两台机器重新同步。如果您注意到两个画面明显偏离，请使用它。锁步 中不需要（也不显示），因为锁步本就不会偏离。
  • 断开连接：结束两台设备的联机对战会话并返回正常的离线游戏。

提示

  • 稳定的 Wi-Fi 连接远比原始速度重要 — 稳定的 2.4GHz 链路通常比拥塞的 5GHz 更好。
  • 会话期间保持两台设备屏幕开启且应用在前台；将其切到后台可能会暂停连接。
  • 如果回滚在某个游戏中感觉卡顿，请尝试将该会话切换到锁步 — 某些硬件（见上文） 根本不适合回滚。
